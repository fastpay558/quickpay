* 发起支付

接口路径：POST：http://{{域名}}/prod-api/service/api/pay

接口说明：调用此接口发起支付请求，返回订单信息以及支付链接信息  

调用参数：

参数名 类型 必填 说明 示例
mchid String 是 商户ID F00001
timeStamp Long 是 请求时间 20250225224505
sign String 是 签名 95d9e2452ce3f6dc39c2ae38622fccb6
channelNo String 是 通道编码 1001
mchTradeNo String 是 商户订单号 FN25ZX02YQ224935PJN3QA
amount Integer 是 交易金额；单位：分 30000
notifyUrl String 是 支付结果回调地址 http://www.xxxx.com/order/notify
payIp String 支付用户ip，传递该参数用于风控控制以提高成功率 111.23.121.367
userFirstName String 用户姓 张
userLastName String 用户名 三
mobile String 手机号 18888888888
email String 邮箱 zhangsan33@gmail.com
remark String 订单备注 这个是订单备注
callbackInfo String 回调传回数据 {"attach":xxx}


响应示例：

{
    "code": 0,
    "msg": "下单成功",
    "time": "20250225224714",
    "data": {
        "mchid": "test",
        "mchTradeNo": "FN25ZX02YQ224935PJN3QA",
        "outTradeNo": "FT0225224714Mq6m8d",
        "channelNo": "1001",
        "amount": 20000,
        "realAmount": 20000,
        "status": 0,
        "statusMsg": "待付款",
        "payUrl": "http://xxxxxxxx/webpage/qrcode?picture=bdfd98e1-aea7-4668-8730-bb7136b6c59a",
        "expireTime": 20250225225214,
        "createTime": "20250225224714",
        "sign": "a434f4fcfdb2f0c698b3a0032f0ce1ef"
    }
}


响应描述： 如果出现异常情况，data 将不会返回，可以通过 code == 0 判断  

参数名 类型 必填 说明 示例
time String 是 返回时间 20250225224714
code String 是 响应标识 0
msg String 是 响应描述 下单成功
data Object 是 响应数据
data >> mchid String 是 商户ID test
data >> mchTradeNo String 是 商户订单号 FN25ZX02YQ224935PJN3QA
data >> outTradeNo String 是 系统订单号 FT0226122721o40PsF
data >> channelNo String 是 通道编码 1001
data >> amount Integer 是 交易金额；单位：分 30000
data >> realAmount Integer 实际支付金额；单位：分 30000
data >> status Integer 是 订单状态；见附录 0
data >> statusMsg Integer 是 订单状态；见附录 待付款
data >> payUrl String 是 付款链接 http://xxxxxxxx/webpage/qrcode?picture=bdfd98e1-aea7-4668-8730-bb7136b6c59a
data >> expireTime Integer 是 链接过期时间 20250226122721
data >> createTime Integer 是 链接创建时间 20250226122721
data >> sign String 是 Data 加签 a434f4fcfdb2f0c698b3a0032f0ce1ef


订单状态：

       0：待付款

       1：付款成功

       2：付款失败

       3：付款超时

       4：补单成功
