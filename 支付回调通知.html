* 支付结果回调通知

接口路径：POST：商户发起支付时传递的notifyUrl地址  

请求方式：Content-Type: application/json  

接口说明：当用户支付成功时，平台将支付结果回调给商户系统；商户收到回调后，需返回 【 { "code": 0 }  】以表示确认收到回调且数据符合预期  

状态说明：收到回调通知时要判断payStatus=1表示订单支付成功，并判断realAmount是否与订单金额一致，防止客户修改金额支付。

重试机制：如果平台未收到 响应标识，会每隔 5秒钟 重新回调，若连续 5 次 回调失败，将不会再尝试自动回调  

调用参数：

参数名 类型 必填 说明 示例
mchId String 是 商户ID test
mchOrderNo String 是 商户订单号 FN25ZX02YQ224935PJN3QA
outOrderNo String 是 系统订单号 FT0226122721o40PsF
payStatus Integer 是 订单状态；只有 1 表示支付成功 0
amount Integer 是 发起支付的金额；单位：分 20000
realAmount Integer 订单实际支付的金额，需要取该字段作为实际支付金额 20000
payTime String 支付成功时返回，订单支付时间；yyyyMMddHHmmss 20250257120200
callbackInfo String 回调信息，商户下单时传入 {"attach":xxx}
errorMsg String 订单附属信息 管理员补单，订单异常信息
timeStamp Long 是 请求时间戳；yyyyMMddHHmmss 20250257120200
sign String 是 请求参数签名 acb4a97c3eedc80c0a1522c6a6522667


订单状态：

       0：待付款

       1：付款成功

       2：付款失败

       3：付款超时

       4：补单成功
