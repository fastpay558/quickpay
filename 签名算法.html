* 签名算法

在调用接口前，需要对调用参数进行签名。签名算法如下：

1：筛选：获取所有不为空的请求参数。

2：排序：将筛选的参数按照第一个字符的键值ASCII码递增排序（字母升序排序），如果遇到相同字符则按照第二个字符的键值ASCII码递增排序，以此类推。

3：拼接：将排序后的参数与其对应值，组合成“参数=参数值”的格式，并且把这些参数用 & 字符连接起来，此时生成的字符串为参数字符串。      

4：秘钥：将商户秘钥拼接在第3步中的参数字符串后面，此时生成的字符串为待签名字符串。

5：签名：使用MD5算法计算第4步中的待签名字符串摘要(不区分大小写)，将此摘要以sign参数一并传递给接口。


例如，某POST接口调用时参数如下，商户秘钥假设为：kys-IPuCLXXG6w-leKkrD4ObK

{
    "mchOrderNo": "F00001",
    "mchid": "test",
    "channelNo": 1001,
    "amount": 30000,
    "notifyUrl": "http://www.xxx.com/url"
}


1：排序拼接后的参数：

amount=30000&channelNo=1001&mchTradeNo=F00001&mchid=test&notifyUrl=http://www.xxx.com/url

2：拼接秘钥后的参数：

amount=30000&channelNo=1001&mchTradeNo=F00001&mchid=test¬ifyUrl=http://www.xxx.com/url&key=Az5PE8dl59v6SLuwgETHw7ndqw7kO7t6

3：使用MD5算法计算出的签名为：

95d9e2452ce3f6dc39c2ae38622fccb6

4：接口调用时，传递的Body参数为：

{
  "mchOrderNo": "F00001",
  "mchid": "test",
  "channelNo": 1001,
  "amount": 30000,
  "notifyUrl": "http://www.xxx.com/url",
  "sign": "95d9e2452ce3f6dc39c2ae38622fccb6"
}
